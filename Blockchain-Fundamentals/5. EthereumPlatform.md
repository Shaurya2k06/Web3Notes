# Ethereum Platform

## Introduction to Ethereum

Ethereum is a decentralized, open-source blockchain platform that enables developers to build and deploy smart contracts and decentralized applications (dApps). Unlike Bitcoin, which focuses primarily on peer-to-peer digital currency, Ethereum provides a programmable blockchain that can execute arbitrary code.

### History and Vision

**Proposed**: December 2013 by Vitalik Buterin
**Launched**: July 30, 2015
**Vision**: "World Computer" - a global, decentralized platform for applications

**Key Milestones:**
- 2015: Frontier (Genesis)
- 2016: Homestead, The DAO hack, Ethereum Classic fork
- 2017: Byzantium, ICO boom
- 2019: Istanbul
- 2020: DeFi Summer, Beacon Chain launch
- 2021: London (EIP-1559)
- 2022: The Merge (PoW → PoS)
- 2023-2024: Shapella, Dencun upgrades

### Ethereum vs Bitcoin

| Aspect | Bitcoin | Ethereum |
|--------|---------|----------|
| **Purpose** | Digital currency | Programmable platform |
| **Block Time** | ~10 minutes | ~12 seconds |
| **Supply** | 21M cap | No hard cap (low inflation) |
| **Scripting** | Limited (Bitcoin Script) | Turing-complete (Solidity) |
| **Consensus** | PoW | PoS (post-Merge) |
| **Primary Use** | Store of value, payments | Smart contracts, dApps, DeFi |

## Ethereum Architecture

### 1. Ethereum Virtual Machine (EVM)

The EVM is a decentralized computer that executes smart contracts across all nodes in the network.

**Key Features:**
- **Turing Complete**: Can execute any computable function
- **Deterministic**: Same input always produces same output
- **Isolated**: Sandboxed environment for security
- **Stack-Based**: 256-bit word size, 1024 stack depth

**EVM Architecture:**
```
┌─────────────────────────────────────┐
│         Smart Contract Code         │
│         (Solidity, Vyper)          │
└──────────────┬──────────────────────┘
               │ Compile
               ▼
┌─────────────────────────────────────┐
│          Bytecode (EVM)             │
│    (Low-level instructions)         │
└──────────────┬──────────────────────┘
               │ Execute
               ▼
┌─────────────────────────────────────┐
│    Ethereum Virtual Machine         │
│  ┌───────────────────────────────┐  │
│  │  Stack (1024 items max)       │  │
│  ├───────────────────────────────┤  │
│  │  Memory (volatile)            │  │
│  ├───────────────────────────────┤  │
│  │  Storage (persistent)         │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 2. Accounts

Ethereum has two types of accounts:

#### Externally Owned Accounts (EOAs)

Controlled by private keys, used by humans.

**Properties:**
- Has ETH balance
- Can send transactions
- No code
- Controlled by private key

```python
class EOA:
    def __init__(self, private_key):
        self.private_key = private_key
        self.address = self.derive_address(private_key)
        self.balance = 0
        self.nonce = 0  # Transaction count
    
    def derive_address(self, private_key):
        # Simplified address derivation
        import hashlib
        public_key = f"public_key_from_{private_key}"
        address = hashlib.sha256(public_key.encode()).hexdigest()[:40]
        return f"0x{address}"
    
    def send_transaction(self, to, value, data=""):
        self.nonce += 1
        return {
            'from': self.address,
            'to': to,
            'value': value,
            'nonce': self.nonce,
            'data': data
        }
```

#### Contract Accounts

Controlled by smart contract code.

**Properties:**
- Has ETH balance
- Has code
- Can store data
- Executes when triggered by transaction

```solidity
// Example Contract Account
contract SimpleWallet {
    address public owner;
    uint256 public balance;
    
    constructor() {
        owner = msg.sender;
    }
    
    receive() external payable {
        balance += msg.value;
    }
    
    function withdraw(uint256 amount) public {
        require(msg.sender == owner, "Not owner");
        require(balance >= amount, "Insufficient balance");
        
        balance -= amount;
        payable(owner).transfer(amount);
    }
}
```

### 3. State and Storage

Ethereum maintains global state: mapping of addresses to account states.

**State Structure:**
```
Global State
├── Address 1
│   ├── Balance: 10 ETH
│   ├── Nonce: 5
│   ├── Code: 0x... (if contract)
│   └── Storage: {key: value} (if contract)
├── Address 2
│   ├── Balance: 2.5 ETH
│   ├── Nonce: 12
│   └── ...
└── ...
```

**State Transitions:**
```python
class EthereumState:
    def __init__(self):
        self.accounts = {}
    
    def get_account(self, address):
        if address not in self.accounts:
            self.accounts[address] = {
                'balance': 0,
                'nonce': 0,
                'code': None,
                'storage': {}
            }
        return self.accounts[address]
    
    def transfer(self, from_addr, to_addr, amount):
        """Execute ETH transfer"""
        from_account = self.get_account(from_addr)
        to_account = self.get_account(to_addr)
        
        if from_account['balance'] < amount:
            raise Exception("Insufficient balance")
        
        from_account['balance'] -= amount
        from_account['nonce'] += 1
        to_account['balance'] += amount
    
    def deploy_contract(self, deployer, code, initial_balance=0):
        """Deploy smart contract"""
        deployer_account = self.get_account(deployer)
        
        # Generate contract address
        import hashlib
        contract_address = hashlib.sha256(
            f"{deployer}{deployer_account['nonce']}".encode()
        ).hexdigest()[:40]
        contract_address = f"0x{contract_address}"
        
        # Create contract account
        self.accounts[contract_address] = {
            'balance': initial_balance,
            'nonce': 0,
            'code': code,
            'storage': {}
        }
        
        deployer_account['nonce'] += 1
        return contract_address

# Example usage
state = EthereumState()

# Create EOA
alice = "0xAlice"
state.get_account(alice)['balance'] = 100

# Transfer ETH
state.transfer(alice, "0xBob", 10)

# Deploy contract
contract_addr = state.deploy_contract(alice, "contract_bytecode")
print(f"Contract deployed at: {contract_addr}")
```

### 4. Transactions

Transactions are signed messages that trigger state changes.

**Transaction Fields:**
```python
class Transaction:
    def __init__(self, nonce, gas_price, gas_limit, to, value, data, v, r, s):
        self.nonce = nonce          # Transaction count from sender
        self.gas_price = gas_price  # Wei per gas unit
        self.gas_limit = gas_limit  # Maximum gas to use
        self.to = to                # Recipient address (None for contract creation)
        self.value = value          # ETH amount to send
        self.data = data            # Contract call data or init code
        self.v = v                  # Signature component
        self.r = r                  # Signature component
        self.s = s                  # Signature component
    
    def hash(self):
        """Calculate transaction hash"""
        import hashlib
        tx_data = f"{self.nonce}{self.gas_price}{self.gas_limit}{self.to}{self.value}{self.data}"
        return hashlib.sha256(tx_data.encode()).hexdigest()
    
    def recover_sender(self):
        """Recover sender address from signature"""
        # Simplified - real implementation uses ECDSA recovery
        return "0xSenderAddress"

# Example: Create and sign transaction
```

## Gas and Fees

### Gas Mechanism

Gas measures computational work required to execute operations.

**Why Gas?**
1. Prevents infinite loops
2. Compensates validators
3. Prioritizes transactions
4. Prevents spam

**Gas Costs (examples):**
```python
GAS_COSTS = {
    'ADD': 3,           # Addition
    'MUL': 5,           # Multiplication
    'SUB': 3,           # Subtraction
    'DIV': 5,           # Division
    'SLOAD': 2100,      # Load from storage
    'SSTORE': 20000,    # Store to storage (new)
    'SSTORE_UPDATE': 5000,  # Update storage
    'CALL': 700,        # Call another contract
    'CREATE': 32000,    # Create contract
    'TRANSFER': 21000   # Simple ETH transfer
}

def calculate_gas(operations):
    """Calculate total gas for operations"""
    total = 0
    for op in operations:
        total += GAS_COSTS.get(op, 0)
    return total

# Example: Simple transfer
transfer_gas = calculate_gas(['TRANSFER'])
print(f"ETH Transfer Gas: {transfer_gas}")

# Example: Contract interaction
contract_call_gas = calculate_gas(['CALL', 'SLOAD', 'SLOAD', 'ADD', 'SSTORE'])
print(f"Contract Call Gas: {contract_call_gas}")
```

### EIP-1559: Fee Market Reform

Introduced in London upgrade (August 2021).

**New Fee Structure:**
```
Total Fee = (Base Fee + Priority Fee) × Gas Used
```

**Components:**
- **Base Fee**: Algorithmically determined, burned
- **Priority Fee (Tip)**: Goes to validator, incentivizes inclusion
- **Max Fee**: Maximum user willing to pay

```python
class EIP1559Transaction:
    def __init__(self, max_fee_per_gas, max_priority_fee_per_gas, gas_limit):
        self.max_fee_per_gas = max_fee_per_gas
        self.max_priority_fee_per_gas = max_priority_fee_per_gas
        self.gas_limit = gas_limit
    
    def calculate_fee(self, base_fee, gas_used):
        """Calculate actual transaction fee"""
        # Effective gas price
        effective_gas_price = min(
            self.max_fee_per_gas,
            base_fee + self.max_priority_fee_per_gas
        )
        
        # Total fee
        total_fee = effective_gas_price * gas_used
        
        # Fee breakdown
        base_fee_paid = base_fee * gas_used  # Burned
        priority_fee_paid = (effective_gas_price - base_fee) * gas_used  # To validator
        
        # Refund
        max_possible_fee = self.max_fee_per_gas * gas_used
        refund = max_possible_fee - total_fee
        
        return {
            'total_fee': total_fee,
            'base_fee_burned': base_fee_paid,
            'priority_fee_to_validator': priority_fee_paid,
            'refund': refund
        }

# Example
tx = EIP1559Transaction(
    max_fee_per_gas=100,  # Gwei
    max_priority_fee_per_gas=2,  # Gwei
    gas_limit=21000
)

base_fee = 50  # Current base fee in Gwei
gas_used = 21000

result = tx.calculate_fee(base_fee, gas_used)
print(f"Total Fee: {result['total_fee']} Gwei")
print(f"Burned: {result['base_fee_burned']} Gwei")
print(f"To Validator: {result['priority_fee_to_validator']} Gwei")
print(f"Refund: {result['refund']} Gwei")
```

**Base Fee Adjustment:**
```python
def calculate_next_base_fee(current_base_fee, gas_used, gas_target):
    """
    Base fee increases if block is >50% full, decreases if <50% full
    """
    if gas_used == gas_target:
        return current_base_fee
    elif gas_used > gas_target:
        # Increase by up to 12.5% per block
        delta = current_base_fee * (gas_used - gas_target) // gas_target // 8
        return current_base_fee + max(delta, 1)
    else:
        # Decrease by up to 12.5% per block
        delta = current_base_fee * (gas_target - gas_used) // gas_target // 8
        return current_base_fee - delta

# Example: Block usage affects base fee
current_base = 50  # Gwei
gas_target = 15_000_000  # 15M gas target per block

# Full block (30M gas used)
next_base_full = calculate_next_base_fee(current_base, 30_000_000, gas_target)
print(f"After full block: {next_base_full} Gwei")

# Half-full block (15M gas used)
next_base_half = calculate_next_base_fee(current_base, 15_000_000, gas_target)
print(f"After half-full block: {next_base_half} Gwei")

# Empty block (0 gas used)
next_base_empty = calculate_next_base_fee(current_base, 0, gas_target)
print(f"After empty block: {next_base_empty} Gwei")
```

## Ethereum Upgrades

### The Merge (September 15, 2022)

Transition from Proof of Work to Proof of Stake.

**Impact:**
```
Energy Consumption: -99.95%
ETH Issuance: -88%
Block Time: ~12s (unchanged)
Finality: Added (~13 minutes)
```

**Staking Requirements:**
- Minimum: 32 ETH
- Rewards: ~4-5% APR
- Penalties: Slashing for malicious behavior

```python
class EthereumStaking:
    def __init__(self):
        self.validators = {}
        self.total_staked = 0
        self.annual_issuance_rate = 0.04  # 4% APR
    
    def stake(self, validator_address, amount):
        """Stake ETH to become validator"""
        if amount < 32:
            return False, "Minimum 32 ETH required"
        
        self.validators[validator_address] = {
            'stake': amount,
            'rewards': 0,
            'active': True
        }
        self.total_staked += amount
        return True, f"Staked {amount} ETH"
    
    def calculate_rewards(self, validator_address, days):
        """Calculate staking rewards"""
        if validator_address not in self.validators:
            return 0
        
        stake = self.validators[validator_address]['stake']
        daily_rate = self.annual_issuance_rate / 365
        rewards = stake * daily_rate * days
        
        return rewards
    
    def slash(self, validator_address, penalty_eth):
        """Slash validator for malicious behavior"""
        if validator_address not in self.validators:
            return False
        
        validator = self.validators[validator_address]
        validator['stake'] -= penalty_eth
        self.total_staked -= penalty_eth
        
        if validator['stake'] < 32:
            validator['active'] = False
        
        return True

# Example
staking = EthereumStaking()

# Alice stakes 32 ETH
staking.stake("0xAlice", 32)

# Calculate rewards after 365 days
rewards = staking.calculate_rewards("0xAlice", 365)
print(f"Annual Rewards: {rewards:.4f} ETH ({(rewards/32)*100:.2f}%)")
```

**Features:**

### Dencun Upgrade (March 2024)

Introduced Proto-Danksharding (EIP-4844).

**Features:**
- Blob transactions for Layer 2 data
- Reduced L2 costs by ~90%
- Foundation for full Danksharding

```python

        self.blob_data = blob_data  # Up to 128 KB
        self.blob_gas_price = self.calculate_blob_gas_price()

        """Blob gas pricing (separate from execution gas)"""
        # Simplified - real implementation uses exponential pricing
        target_blobs_per_block = 3
        max_blobs_per_block = 6
        
        # Much cheaper than calldata
        return 1  # Wei per blob gas unit
    
    def get_cost_savings(self):
        """Compare to pre-EIP-4844 calldata cost"""
        calldata_cost = len(self.blob_data) * 16  # 16 gas per byte
```

## Ethereum Roadmap

### The Surge: Scalability
- **Goal**: 100,000+ TPS with rollups
- **Solutions**: Danksharding, rollup improvements

### The Scourge: Censorship Resistance
- **Goal**: Prevent MEV exploitation
- **Solutions**: PBS (Proposer-Builder Separation), encrypted mempools

### The Verge: Verification
- **Goal**: Make running nodes easier
- **Solutions**: Verkle trees, stateless clients

### The Purge: Simplification
- **Goal**: Reduce node storage requirements
- **Solutions**: State expiry, history expiry

### The Splurge: Everything Else
- **Goal**: Various improvements
- **Solutions**: Account abstraction, EVM improvements
