# Cryptography and Blockchain Security

## Introduction to Cryptography

Cryptography is the foundation of blockchain security, enabling trustless transactions, data integrity, and user authentication without central authorities. It combines mathematical principles with computer science to create secure communication systems.

### What is Cryptography?

**Definition**: The practice and study of techniques for secure communication in the presence of adversaries.

**Core Goals:**
1. **Confidentiality**: Only authorized parties can read the data
2. **Integrity**: Data cannot be altered without detection
3. **Authentication**: Verify the identity of parties
4. **Non-repudiation**: Sender cannot deny sending a message

## Cryptographic Hash Functions

### What is a Hash Function?

A hash function takes an input of any size and produces a fixed-size output (hash/digest).

**Properties of Cryptographic Hash Functions:**

#### 1. Deterministic
Same input always produces the same output.

```python
import hashlib

message = "Hello, Blockchain!"
hash1 = hashlib.sha256(message.encode()).hexdigest()
hash2 = hashlib.sha256(message.encode()).hexdigest()

print(f"Hash 1: {hash1}")
print(f"Hash 2: {hash2}")
print(f"Equal: {hash1 == hash2}")  # True
```

#### 2. Quick Computation
Fast to calculate hash from input.

```python
import hashlib
import time

# Hash a large file quickly
def hash_file(filename):
    start = time.time()
    hasher = hashlib.sha256()
    
    with open(filename, 'rb') as f:
        while chunk := f.read(8192):
            hasher.update(chunk)
    
    end = time.time()
    return hasher.hexdigest(), end - start

# Even large files hash in milliseconds
```

#### 3. Avalanche Effect
Small change in input drastically changes output.

```python
import hashlib

input1 = "Hello, Blockchain!"
input2 = "Hello, Blockchain?"  # Changed ! to ?

hash1 = hashlib.sha256(input1.encode()).hexdigest()
hash2 = hashlib.sha256(input2.encode()).hexdigest()

print(f"Input 1: {input1}")
print(f"Hash 1:  {hash1}")
print(f"\nInput 2: {input2}")
print(f"Hash 2:  {hash2}")
print(f"\nCompletely different despite minimal input change!")
```


#### 4. One-Way Function (Preimage Resistance)
Impossible to reverse - cannot find input from output.

```python
# Given a hash, you cannot determine the original input
hash_value = "2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824"

# No function exists to do this:
# original_input = reverse_hash(hash_value)  # IMPOSSIBLE!

# Only way to find input is brute force (try all possibilities)
```

#### 5. Collision Resistance
Nearly impossible to find two different inputs with the same hash.

**Birthday Paradox**: For a hash function with n-bit output, expect collision after ~2^(n/2) attempts.

For SHA-256 (256 bits):
- 2^128 attempts needed for collision
- More than atoms in the universe
- Computationally infeasible

### Common Hash Functions in Blockchain

#### SHA-256 (Secure Hash Algorithm 256-bit)

**Used by**: Bitcoin, many other blockchains

**Properties:**
- Output: 256 bits (64 hexadecimal characters)
- Designed by NSA, published in 2001
- Part of SHA-2 family

```python
import hashlib

def sha256_hash(data):
    return hashlib.sha256(data.encode()).hexdigest()

# Example
message = "Bitcoin uses SHA-256"
hash_output = sha256_hash(message)
print(f"SHA-256: {hash_output}")
# Output: 64 character hexadecimal string
```

#### Keccak-256

**Used by**: Ethereum

**Properties:**
- Output: 256 bits
- Winner of SHA-3 competition
- Different from official SHA-3 (Ethereum uses pre-NIST version)

```python

def keccak256_hash(data):
    k = keccak.new(digest_bits=256)
    k.update(data.encode())
    return k.hexdigest()

# Example

```

#### RIPEMD-160

**Used by**: Bitcoin (for address generation)

**Properties:**
- Output: 160 bits
- Often used after SHA-256 in Bitcoin

```python
import hashlib

def hash160(data):
    """Bitcoin's HASH160: RIPEMD160(SHA256(data))"""
    sha = hashlib.sha256(data.encode()).digest()
    ripe = hashlib.new('ripemd160', sha).hexdigest()
    return ripe

# Used in Bitcoin address generation
```

### Hash Function Applications in Blockchain

#### 1. Block Linking
Each block contains hash of previous block, creating immutable chain.

```python
class Block:
    def __init__(self, index, data, previous_hash):
        self.index = index
        self.data = data
        self.previous_hash = previous_hash
        self.hash = self.calculate_hash()

# Create blockchain
```

#### 2. Proof of Work
Miners must find hash meeting difficulty target.

```python
import hashlib
import time

def proof_of_work(block_data, difficulty):
    """
    Find nonce that produces hash with 'difficulty' leading zeros
    """
    target = '0' * difficulty
    nonce = 0
    start_time = time.time()
    
    while True:
        text = f"{block_data}{nonce}"
        hash_result = hashlib.sha256(text.encode()).hexdigest()
        
        if hash_result.startswith(target):
            end_time = time.time()
            return {
                'nonce': nonce,
                'hash': hash_result,
                'time': end_time - start_time,
                'attempts': nonce + 1
            }
        
        nonce += 1

# Example: Mine with difficulty 5
result = proof_of_work("Block #12345", difficulty=5)
print(f"Found nonce: {result['nonce']}")
print(f"Hash: {result['hash']}")
print(f"Time: {result['time']:.2f}s")
print(f"Attempts: {result['attempts']:,}")
```

#### 3. Address Generation
Public keys are hashed to create addresses.

```python
import hashlib

def generate_bitcoin_address(public_key):
    """
    Simplified Bitcoin address generation
    """
    # Step 1: SHA-256 hash of public key
    sha256_hash = hashlib.sha256(public_key.encode()).digest()
    
    # Step 2: RIPEMD-160 hash
    ripemd160_hash = hashlib.new('ripemd160', sha256_hash).digest()
    
    # Step 3: Add version byte (0x00 for mainnet)
    versioned = b'\x00' + ripemd160_hash
    
    # Step 4: Double SHA-256 for checksum
    checksum = hashlib.sha256(hashlib.sha256(versioned).digest()).digest()[:4]
    
    # Step 5: Concatenate and encode in Base58
    address_bytes = versioned + checksum
    
    return address_bytes.hex()

# Example
public_key = "04678afdb0fe5548271967f1a67130b7105cd6a828e03909a67962e0ea1f61deb649f6bc3f4cef38c4f35504e51ec112de5c384df7ba0b8d578a4c702b6bf11d5f"
address = generate_bitcoin_address(public_key)
print(f"Bitcoin Address: {address}")
```

#### 4. Merkle Trees
Efficiently verify transactions in a block.

```python
import hashlib

class MerkleTree:
    def __init__(self, transactions):
        self.transactions = transactions
        self.tree = self.build_tree()
    
    def hash(self, data):
        return hashlib.sha256(data.encode()).hexdigest()
    
    def build_tree(self):
        # Start with transaction hashes
        current_level = [self.hash(tx) for tx in self.transactions]
        tree = [current_level]
        
        # Build tree bottom-up
        while len(current_level) > 1:
            next_level = []
            
            # Pair up hashes
            for i in range(0, len(current_level), 2):
                left = current_level[i]
                right = current_level[i + 1] if i + 1 < len(current_level) else left
                
                # Hash the pair
                combined = self.hash(left + right)
                next_level.append(combined)
            
            tree.append(next_level)
            current_level = next_level
        
        return tree
    
    def get_root(self):
        return self.tree[-1][0]
    
    def verify_transaction(self, tx_index, proof):
        """Verify a transaction is in the tree using Merkle proof"""
        current_hash = self.hash(self.transactions[tx_index])
        
        for direction, sibling_hash in proof:
            if direction == 'left':
                current_hash = self.hash(sibling_hash + current_hash)
            else:
                current_hash = self.hash(current_hash + sibling_hash)
        
        return current_hash == self.get_root()

# Example
transactions = [
    "Alice -> Bob: 1 BTC",
    "Charlie -> David: 2 BTC",
    "Eve -> Frank: 0.5 BTC",
    "Grace -> Henry: 3 BTC"
]

merkle_tree = MerkleTree(transactions)
print(f"Merkle Root: {merkle_tree.get_root()}")
```

## Public Key Cryptography

### Asymmetric Encryption

Unlike symmetric encryption (same key for encryption/decryption), asymmetric uses key pairs.

**Key Pair:**
- **Private Key**: Secret, never shared, used to sign transactions
- **Public Key**: Shared publicly, used to verify signatures and receive funds

```
Private Key → Public Key → Address
(secret)      (shareable)  (blockchain identifier)
```

### Elliptic Curve Cryptography (ECC)

Most blockchains use ECC for digital signatures due to strong security with smaller key sizes.

**Advantages over RSA:**
- Smaller keys (256-bit ECC ≈ 3072-bit RSA security)
- Faster computation
- Less bandwidth
- Lower storage requirements

#### ECDSA (Elliptic Curve Digital Signature Algorithm)

**Used by**: Bitcoin, Ethereum (pre-EIP-2537)

**Curve**: secp256k1

```python
from cryptography.hazmat.primitives.asymmetric import ec
from cryptography.hazmat.primitives import hashes

# Generate key pair
private_key = ec.generate_private_key(ec.SECP256K1())
public_key = private_key.public_key()

# Sign a message
message = b"Transfer 1 BTC to Bob"
signature = private_key.sign(
    message,
    ec.ECDSA(hashes.SHA256())
)

# Verify signature
try:
    public_key.verify(
        signature,
        message,
        ec.ECDSA(hashes.SHA256())
    )
    print("✓ Signature is valid!")
except:
    print("✗ Signature is invalid!")
```

#### EdDSA (Edwards-curve Digital Signature Algorithm)

**Used by**: Solana, Cardano, Polkadot

**Curve**: Ed25519

**Advantages:**
- Faster than ECDSA
- Deterministic signatures (no random nonce)
- Resistant to side-channel attacks

```python
from cryptography.hazmat.primitives.asymmetric import ed25519

# Generate key pair
private_key = ed25519.Ed25519PrivateKey.generate()
public_key = private_key.public_key()

# Sign message
message = b"Transfer 100 SOL to Alice"
signature = private_key.sign(message)

# Verify signature
try:
    public_key.verify(signature, message)
    print("✓ Signature is valid!")
except:
    print("✗ Signature is invalid!")
```

### Digital Signatures in Blockchain

#### How Transactions are Signed

```python
class Transaction:
    def __init__(self, sender, recipient, amount, private_key):
        self.sender = sender
        self.recipient = recipient
        self.amount = amount
        self.signature = self.sign_transaction(private_key)
    
    def get_transaction_data(self):
        return f"{self.sender}{self.recipient}{self.amount}"
    
    def sign_transaction(self, private_key):
        """Sign transaction with private key"""
        import hashlib
        from cryptography.hazmat.primitives.asymmetric import ec
        from cryptography.hazmat.primitives import hashes
        
        # Hash transaction data
        tx_data = self.get_transaction_data()
        tx_hash = hashlib.sha256(tx_data.encode()).digest()
        
        # Sign hash
        signature = private_key.sign(
            tx_hash,
            ec.ECDSA(hashes.SHA256())
        )
        
        return signature
    
    def verify_signature(self, public_key):
        """Verify transaction signature"""
        import hashlib
        from cryptography.hazmat.primitives.asymmetric import ec
        from cryptography.hazmat.primitives import hashes
        
        tx_data = self.get_transaction_data()
        tx_hash = hashlib.sha256(tx_data.encode()).digest()
        
        try:
            public_key.verify(
                self.signature,
                tx_hash,
                ec.ECDSA(hashes.SHA256())
            )
            return True
        except:
            return False

# Example usage
from cryptography.hazmat.primitives.asymmetric import ec

# Alice creates and signs a transaction
alice_private_key = ec.generate_private_key(ec.SECP256K1())
alice_public_key = alice_private_key.public_key()

tx = Transaction(
    sender="Alice",
    recipient="Bob",
    amount=1.5,
    private_key=alice_private_key
)

# Anyone can verify the signature
is_valid = tx.verify_signature(alice_public_key)
print(f"Transaction valid: {is_valid}")
```


## Blockchain Security Mechanisms

### 1. Consensus Security

#### 51% Attack

**Definition**: When an entity controls >50% of network hash power (PoW) or stake (PoS), they can manipulate the blockchain.

**What attackers can do:**
- Double-spend coins
- Prevent transaction confirmations
- Reverse their own transactions

**What attackers CANNOT do:**
- Steal coins from other addresses
- Change consensus rules
- Create coins out of thin air

**Cost Analysis:**
```python
def calculate_51_attack_cost(network_hashrate, hardware_efficiency, electricity_cost, duration_hours):
    """
    Calculate cost of 51% attack on PoW network
    
    network_hashrate: TH/s (terahashes per second)
    hardware_efficiency: J/TH (joules per terahash)
    electricity_cost: $/kWh
    duration_hours: hours to maintain attack
    """
    # Need 51% of network hashrate
    required_hashrate = network_hashrate * 0.51
    
    # Power consumption in kW
    power_kw = (required_hashrate * hardware_efficiency) / 3600
    
    # Electricity cost
    electricity_total = power_kw * duration_hours * electricity_cost
    
    # Hardware cost (approximate)
    hardware_cost_per_th = 50  # $50 per TH/s
    hardware_total = required_hashrate * hardware_cost_per_th
    
    return {
        'required_hashrate_th': required_hashrate,
        'power_consumption_kw': power_kw,
        'electricity_cost': electricity_total,
        'hardware_cost': hardware_total,
        'total_cost': electricity_total + hardware_total
    }

# Example: Bitcoin network
bitcoin_hashrate = 400_000_000  # 400 EH/s = 400,000,000 TH/s
cost = calculate_51_attack_cost(
    network_hashrate=bitcoin_hashrate,
    hardware_efficiency=30,  # Modern ASIC
    electricity_cost=0.10,   # $0.10 per kWh
    duration_hours=24        # 24 hour attack
)

print(f"51% Attack Cost on Bitcoin:")
print(f"Required Hashrate: {cost['required_hashrate_th']:,.0f} TH/s")
print(f"Hardware Cost: ${cost['hardware_cost']:,.0f}")
print(f"Electricity Cost (24h): ${cost['electricity_cost']:,.0f}")
print(f"Total Cost: ${cost['total_cost']:,.0f}")
```

**Defense Mechanisms:**
- High network hashrate/stake makes attacks expensive
- Checkpoint systems
- Delayed finality
- Social consensus (community can reject malicious chain)

#### Sybil Attack

**Definition**: Creating multiple fake identities to gain disproportionate influence.

**Prevention in Blockchain:**
- **PoW**: Requires computational resources (can't fake hashpower)
- **PoS**: Requires capital (can't fake stake)
- **PoA**: Requires verified identity
- **Reputation systems**: Build trust over time

### 2. Smart Contract Security

#### Common Vulnerabilities

**1. Reentrancy Attack**

Exploits recursive calls to drain funds.

```solidity
// VULNERABLE CONTRACT
contract VulnerableBank {
    mapping(address => uint) public balances;
    
    function deposit() public payable {
        balances[msg.sender] += msg.value;
    }
    
    function withdraw(uint amount) public {
        require(balances[msg.sender] >= amount);
        
        // VULNERABILITY: External call before state update
        (bool success, ) = msg.sender.call{value: amount}("");
        require(success);
        
        balances[msg.sender] -= amount;  // State updated AFTER external call
    }
}

// ATTACKER CONTRACT
contract Attacker {
    VulnerableBank public bank;
    
    constructor(address _bank) {
        bank = VulnerableBank(_bank);
    }
    
    function attack() public payable {
        bank.deposit{value: msg.value}();
        bank.withdraw(msg.value);
    }
    
    // Reentrancy: Called when receiving ETH
    receive() external payable {
        if (address(bank).balance >= msg.value) {
            bank.withdraw(msg.value);  // Recursive call!
        }
    }
}

// SECURE CONTRACT (using Checks-Effects-Interactions pattern)
contract SecureBank {
    mapping(address => uint) public balances;
    
    function withdraw(uint amount) public {
        require(balances[msg.sender] >= amount);
        
        // Update state BEFORE external call
        balances[msg.sender] -= amount;
        
        (bool success, ) = msg.sender.call{value: amount}("");
        require(success);
    }
}
```

**2. Integer Overflow/Underflow**

Arithmetic operations exceeding variable limits.

```solidity
// VULNERABLE (Solidity < 0.8.0)
contract VulnerableToken {
    mapping(address => uint8) public balances;
    
    function transfer(address to, uint8 amount) public {
        // If balances[msg.sender] = 0 and amount = 1
        // 0 - 1 = 255 (underflow!)
        balances[msg.sender] -= amount;
        balances[to] += amount;
    }
}

// SECURE (Solidity >= 0.8.0 has built-in overflow checks)
contract SecureToken {
    mapping(address => uint256) public balances;
    
    function transfer(address to, uint256 amount) public {
        require(balances[msg.sender] >= amount, "Insufficient balance");
        balances[msg.sender] -= amount;  // Reverts on underflow
        balances[to] += amount;           // Reverts on overflow
    }
}
```

**3. Front-Running**

Miners/bots see pending transactions and submit their own with higher gas to execute first.

```solidity
// VULNERABLE: Price oracle update
contract VulnerableOracle {
    uint public price;
    
    function updatePrice(uint newPrice) public {
        price = newPrice;
    }
}

// Attacker sees updatePrice transaction in mempool
// Submits trade before price update with higher gas fee
// Profits from price change

// MITIGATION: Commit-Reveal scheme
contract SecureOracle {
    mapping(address => bytes32) public commits;
    uint public price;
    
    // Step 1: Commit hash of price
    function commitPrice(bytes32 hash) public {
        commits[msg.sender] = hash;
    }
    
    // Step 2: Reveal actual price (after commit is mined)
    function revealPrice(uint newPrice, bytes32 salt) public {
        require(keccak256(abi.encodePacked(newPrice, salt)) == commits[msg.sender]);
        price = newPrice;
    }
}
```

**4. Access Control Issues**

Improper permission checks allow unauthorized actions.

```solidity
// VULNERABLE
contract VulnerableWallet {
    address public owner;
    
    function withdraw(uint amount) public {
        // Missing access control!
        payable(msg.sender).transfer(amount);
    }
}

// SECURE
contract SecureWallet {
    address public owner;
    
    modifier onlyOwner() {
        require(msg.sender == owner, "Not authorized");
        _;
    }
    
    function withdraw(uint amount) public onlyOwner {
        payable(owner).transfer(amount);
    }
}
```

#### Smart Contract Security Best Practices

```solidity
// COMPREHENSIVE SECURE CONTRACT EXAMPLE
pragma solidity ^0.8.0;

import "@openzeppelin/contracts/security/ReentrancyGuard.sol";
import "@openzeppelin/contracts/access/Ownable.sol";
import "@openzeppelin/contracts/security/Pausable.sol";

contract SecureVault is ReentrancyGuard, Ownable, Pausable {
    mapping(address => uint256) public balances;
    
    event Deposit(address indexed user, uint256 amount);
    event Withdrawal(address indexed user, uint256 amount);
    
    // 1. Use ReentrancyGuard for external calls
    function deposit() public payable nonReentrant whenNotPaused {
        require(msg.value > 0, "Must deposit something");
        
        balances[msg.sender] += msg.value;
        emit Deposit(msg.sender, msg.value);
    }
    
    // 2. Checks-Effects-Interactions pattern
    function withdraw(uint256 amount) public nonReentrant whenNotPaused {
        // Checks
        require(amount > 0, "Must withdraw something");
        require(balances[msg.sender] >= amount, "Insufficient balance");
        
        // Effects (update state before external call)
        balances[msg.sender] -= amount;
        
        // Interactions (external call last)
        (bool success, ) = payable(msg.sender).call{value: amount}("");
        require(success, "Transfer failed");
        
        emit Withdrawal(msg.sender, amount);
    }
    
    // 3. Emergency pause functionality
    function pause() public onlyOwner {
        _pause();
    }
    
    function unpause() public onlyOwner {
        _unpause();
    }
    
    // 4. Explicit visibility
    function getBalance() public view returns (uint256) {
        return balances[msg.sender];
    }
}
```

### 3. Network Security

#### Eclipse Attack

**Definition**: Isolating a node by controlling all its peer connections.

**Attack Process:**
1. Attacker controls multiple nodes
2. Target node connects only to attacker's nodes
3. Attacker feeds false blockchain data
4. Target accepts invalid transactions

**Defense:**
- Diverse peer selection
- Trusted peer lists
- Regular peer rotation
- Geographic diversity

#### DDoS (Distributed Denial of Service)

**Definition**: Overwhelming network with traffic to disrupt service.

**Blockchain-Specific DDoS:**
- Transaction spam
- Block propagation delays
- Node resource exhaustion

**Mitigation:**
- Transaction fees (economic spam prevention)
- Rate limiting
- Proof-of-Work for requests
- Network redundancy

#### Multi-Signature Wallets

Require multiple signatures to authorize transactions.

```solidity
contract MultiSigWallet {
    address[] public owners;
    uint public required;  // Number of signatures needed
    
    struct Transaction {
        address to;
        uint value;
        bytes data;
        bool executed;
        uint confirmations;
    }
    
    Transaction[] public transactions;
    mapping(uint => mapping(address => bool)) public confirmations;
    
    constructor(address[] memory _owners, uint _required) {
        require(_owners.length >= _required);
        require(_required > 0);
        
        owners = _owners;
        required = _required;
    }
    
    function submitTransaction(address to, uint value, bytes memory data) 
        public 
        returns (uint) 
    {
        uint txId = transactions.length;
        transactions.push(Transaction({
            to: to,
            value: value,
            data: data,
            executed: false,
            confirmations: 0
        }));
        
        return txId;
    }
    
    function confirmTransaction(uint txId) public {
        require(isOwner(msg.sender));
        require(!confirmations[txId][msg.sender]);
        
        confirmations[txId][msg.sender] = true;
        transactions[txId].confirmations++;
        
        if (transactions[txId].confirmations >= required) {
            executeTransaction(txId);
        }
    }
    
    function executeTransaction(uint txId) internal {
        require(!transactions[txId].executed);
        require(transactions[txId].confirmations >= required);
        
        Transaction storage txn = transactions[txId];
        txn.executed = true;
        
        (bool success, ) = txn.to.call{value: txn.value}(txn.data);
        require(success);
    }
    
    function isOwner(address addr) public view returns (bool) {
        for (uint i = 0; i < owners.length; i++) {
            if (owners[i] == addr) return true;
        }
        return false;
    }
}
```

## Advanced Cryptographic Techniques

### 1. Zero-Knowledge Proofs (ZKPs)

Prove knowledge of information without revealing the information itself.

**Example**: Prove you're over 18 without revealing your birthdate.

**Applications:**
- Private transactions (Zcash)
- Scalability (zkRollups)
- Identity verification
- Voting systems

### 2. Ring Signatures

Sign message on behalf of group without revealing which member signed.

**Used by**: Monero for transaction privacy

**Properties:**
- Signer anonymity within group
- Verifiable that someone in group signed
- Cannot determine which member

### 3. Threshold Signatures

Distribute signing power among multiple parties.

**Use Cases:**
- Distributed key generation
- Secure multi-party computation
- Validator committees

### 4. Homomorphic Encryption

Perform computations on encrypted data without decrypting.

**Applications:**
- Private smart contracts
- Confidential transactions
- Secure data analysis
