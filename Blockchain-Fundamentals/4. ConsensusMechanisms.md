# Consensus Mechanisms

## What is Consensus?

In distributed systems, consensus is the process by which a network of nodes agrees on a single version of truth. In blockchain, this means agreeing on the order and validity of transactions without a central authority.

### The Byzantine Generals Problem

**Scenario**: Multiple generals surround a city and must coordinate an attack. They can only communicate via messengers, but some generals may be traitors sending false messages.

**Challenge**: How do loyal generals reach consensus despite traitors?

**Blockchain Solution**: Consensus mechanisms ensure network agreement even when some nodes are malicious or faulty.

```
Traditional Systems: Trust central authority
Blockchain: Mathematical/economic consensus without trust
```

### Requirements for Blockchain Consensus

1. **Agreement**: All honest nodes agree on the same state
2. **Validity**: Only valid transactions are accepted
3. **Termination**: Nodes eventually reach consensus
4. **Fault Tolerance**: System works despite node failures
5. **Censorship Resistance**: No single entity can block transactions

## Proof of Work (PoW)

### Overview

Miners compete to solve computational puzzles to add blocks and earn rewards.

**Used by**: Bitcoin, Ethereum (pre-Merge), Litecoin, Dogecoin, Monero

### How PoW Works

#### 1. Transaction Pool
Pending transactions wait in mempool.

#### 2. Block Creation
Miner selects transactions and creates candidate block.

#### 3. Mining Process
Find nonce that produces hash meeting difficulty target.

```python
import hashlib
import time

class ProofOfWork:
    def __init__(self, difficulty):
        self.difficulty = difficulty
        self.target = '0' * difficulty
    
    def mine_block(self, block_data):
        """
        Find nonce that produces hash with required leading zeros
        """
        nonce = 0
        start_time = time.time()
        
        while True:
            # Combine block data with nonce
            data = f"{block_data}{nonce}"
            hash_result = hashlib.sha256(data.encode()).hexdigest()
            
            # Check if hash meets difficulty target
            if hash_result.startswith(self.target):
                end_time = time.time()
                return {
                    'nonce': nonce,
                    'hash': hash_result,
                    'attempts': nonce + 1,
                    'time': end_time - start_time,
                    'hash_rate': nonce / (end_time - start_time)
                }
            
            nonce += 1
    
    def verify_block(self, block_data, nonce, claimed_hash):
        """
        Verify that nonce produces valid hash
        """
        data = f"{block_data}{nonce}"
        actual_hash = hashlib.sha256(data.encode()).hexdigest()
        
        return (actual_hash == claimed_hash and 
                actual_hash.startswith(self.target))

# Example: Mine blocks with different difficulties
pow = ProofOfWork(difficulty=5)

print("Mining block with difficulty 5...")
result = pow.mine_block("Block #12345: Alice sends 1 BTC to Bob")

print(f"✓ Block mined!")
print(f"  Nonce: {result['nonce']}")
print(f"  Hash: {result['hash']}")
print(f"  Attempts: {result['attempts']:,}")
print(f"  Time: {result['time']:.2f}s")
print(f"  Hash Rate: {result['hash_rate']:,.0f} H/s")

# Verification is instant
is_valid = pow.verify_block(
    "Block #12345: Alice sends 1 BTC to Bob",
    result['nonce'],
    result['hash']
)
print(f"  Valid: {is_valid}")
```


#### 4. Difficulty Adjustment

Network adjusts difficulty to maintain consistent block time.

```python
class DifficultyAdjustment:
    def __init__(self, target_block_time, adjustment_interval):
        self.target_block_time = target_block_time  # seconds
        self.adjustment_interval = adjustment_interval  # blocks
        self.current_difficulty = 4
    
    def adjust_difficulty(self, actual_time_taken):
        """
        Adjust difficulty based on actual vs target time
        """
        expected_time = self.target_block_time * self.adjustment_interval
        
        if actual_time_taken < expected_time:
            # Blocks mined too fast, increase difficulty
            self.current_difficulty += 1
            return "increased"
        elif actual_time_taken > expected_time * 1.5:
            # Blocks mined too slow, decrease difficulty
            self.current_difficulty = max(1, self.current_difficulty - 1)
            return "decreased"
        else:
            return "unchanged"
    
    def get_difficulty(self):
        return self.current_difficulty

# Bitcoin example: Adjusts every 2016 blocks (~2 weeks)
bitcoin_difficulty = DifficultyAdjustment(
    target_block_time=600,  # 10 minutes
    adjustment_interval=2016
)

# Simulate: 2016 blocks mined in 1 week (too fast)
actual_time = 7 * 24 * 60 * 60  # 1 week in seconds
adjustment = bitcoin_difficulty.adjust_difficulty(actual_time)
print(f"Difficulty {adjustment}: {bitcoin_difficulty.get_difficulty()}")
```

### PoW Advantages

✅ **Battle-Tested**: Bitcoin running since 2009
✅ **Highly Secure**: 51% attack extremely expensive
✅ **Truly Decentralized**: Anyone can mine
✅ **Simple to Understand**: Clear economic incentives
✅ **Objective Finality**: Longest chain rule

### PoW Disadvantages

❌ **Energy Intensive**: Bitcoin uses ~150 TWh/year
❌ **Slow Finality**: Multiple confirmations needed
❌ **Scalability Limited**: ~7 TPS for Bitcoin
❌ **Mining Centralization**: ASIC manufacturers, mining pools
❌ **51% Attack Risk**: For smaller networks

### Bitcoin Mining Economics

```python
def calculate_mining_profitability(
    hashrate_th,           # TH/s
    power_consumption_w,   # Watts
    electricity_cost,      # $/kWh
    btc_price,            # USD
    network_hashrate_eh,  # EH/s
    block_reward          # BTC
):
    """
    Calculate daily mining profitability
    """
    # Convert network hashrate to TH/s
    network_hashrate_th = network_hashrate_eh * 1_000_000
    
    # Probability of mining a block
    blocks_per_day = 144  # Bitcoin: 1 block per 10 minutes
    hash_share = hashrate_th / network_hashrate_th
    expected_blocks = blocks_per_day * hash_share
    
    # Revenue
    daily_btc = expected_blocks * block_reward
    daily_revenue = daily_btc * btc_price
    
    # Costs
    daily_kwh = (power_consumption_w / 1000) * 24
    daily_electricity_cost = daily_kwh * electricity_cost
    
    # Profit
    daily_profit = daily_revenue - daily_electricity_cost
    
    return {
        'daily_btc': daily_btc,
        'daily_revenue': daily_revenue,
        'daily_cost': daily_electricity_cost,
        'daily_profit': daily_profit,
        'roi_days': (15000 / daily_profit) if daily_profit > 0 else float('inf')  # Assuming $15k hardware
    }

# Example: Antminer S19 Pro
result = calculate_mining_profitability(
    hashrate_th=110,
    power_consumption_w=3250,
    electricity_cost=0.10,
    btc_price=45000,
    network_hashrate_eh=400,
    block_reward=6.25
```

## Proof of Stake (PoS)

### Overview

Validators stake cryptocurrency as collateral to propose and validate blocks.

**Used by**: Ethereum (post-Merge), Cardano, Polkadot, Solana, Avalanche

### How PoS Works

#### 1. Staking
Validators lock up cryptocurrency as collateral.

```python
class ProofOfStake:
    def __init__(self, minimum_stake):
        self.minimum_stake = minimum_stake
        self.validators = {}
        self.total_stake = 0
    
    def stake(self, validator_address, amount):
        """
        Validator stakes tokens
        """
        if amount < self.minimum_stake:
            return False, "Insufficient stake amount"
        
        if validator_address in self.validators:
            self.validators[validator_address] += amount
        else:
            self.validators[validator_address] = amount
        
        self.total_stake += amount
        return True, f"Staked {amount} tokens"
    
    def unstake(self, validator_address, amount):
        """
        Validator unstakes tokens (with unbonding period)
        """
        if validator_address not in self.validators:
            return False, "Not a validator"
        
        if self.validators[validator_address] < amount:
            return False, "Insufficient staked amount"
        
        self.validators[validator_address] -= amount
        self.total_stake -= amount
        
        if self.validators[validator_address] == 0:
            del self.validators[validator_address]
        
        return True, f"Unstaked {amount} tokens (unbonding period: 7 days)"

# Ethereum example: 32 ETH minimum
eth_pos = ProofOfStake(minimum_stake=32)

# Validators stake
eth_pos.stake("0xAlice", 32)
eth_pos.stake("0xBob", 64)
eth_pos.stake("0xCarol", 128)

print(f"Total Staked: {eth_pos.total_stake} ETH")
print(f"Validators: {len(eth_pos.validators)}")
```

#### 2. Validator Selection

Validators chosen based on stake weight and randomness.

```python
import random
import hashlib

class ValidatorSelection:
    def __init__(self, validators):
        self.validators = validators  # {address: stake}
        self.total_stake = sum(validators.values())
    
    def weighted_random_selection(self):
        """
        Select validator with probability proportional to stake
        """
        selection_point = random.uniform(0, self.total_stake)
        current_sum = 0
        
        for address, stake in self.validators.items():
            current_sum += stake
            if current_sum >= selection_point:
                return address
        
        return list(self.validators.keys())[-1]
    
    def vrf_selection(self, seed, slot):
        """
        Verifiable Random Function for deterministic but unpredictable selection
        """
        # Simplified VRF (real implementation uses cryptographic VRF)
        combined = f"{seed}{slot}"
        hash_value = int(hashlib.sha256(combined.encode()).hexdigest(), 16)
        selection_point = (hash_value % self.total_stake)
        
        current_sum = 0
        for address, stake in self.validators.items():
            current_sum += stake
            if current_sum >= selection_point:
                return address
        
        return list(self.validators.keys())[-1]

# Example
validators = {
    "0xAlice": 32,
    "0xBob": 64,
    "0xCarol": 128,
    "0xDave": 256
}

selector = ValidatorSelection(validators)

# Simulate 1000 selections
selections = {}
for i in range(1000):
    validator = selector.vrf_selection("random_seed", i)
    selections[validator] = selections.get(validator, 0) + 1

print("Validator Selection Distribution (1000 slots):")
for validator, count in sorted(selections.items(), key=lambda x: x[1], reverse=True):
    percentage = (count / 1000) * 100
    stake = validators[validator]
    expected = (stake / sum(validators.values())) * 100
    print(f"{validator}: {count} ({percentage:.1f}% actual, {expected:.1f}% expected)")
```

#### 3. Block Proposal and Attestation

Selected validator proposes block, others attest to validity.

```python
class BlockProposalAndAttestation:
    def __init__(self, validators):
        self.validators = validators
        self.attestations = {}
    
    def propose_block(self, proposer, block_data):
        """
        Proposer creates and broadcasts block
        """
        block = {
            'proposer': proposer,
            'data': block_data,
            'attestations': []
        }
        return block
    
    def attest_block(self, block, attester):
        """
        Validator attests to block validity
        """
        if attester not in self.validators:
            return False, "Not a validator"
        
        attestation = {
            'attester': attester,
            'block_hash': hash(str(block)),
            'signature': f"sig_{attester}"
        }
        
        block['attestations'].append(attestation)
        return True, "Attestation added"
    
    def finalize_block(self, block, threshold=0.67):
        """
        Block finalized when >2/3 of stake attests
        """
        attesting_stake = sum(
            self.validators[att['attester']] 
            for att in block['attestations']
            if att['attester'] in self.validators
        )
        
        total_stake = sum(self.validators.values())
        attestation_percentage = attesting_stake / total_stake
        
        if attestation_percentage >= threshold:
            return True, f"Block finalized ({attestation_percentage:.1%} attested)"
        else:
            return False, f"Insufficient attestations ({attestation_percentage:.1%})"

# Example
validators = {"0xAlice": 32, "0xBob": 64, "0xCarol": 128, "0xDave": 256}
consensus = BlockProposalAndAttestation(validators)

# Alice proposes block
block = consensus.propose_block("0xAlice", "Block #12345")

# Validators attest
consensus.attest_block(block, "0xBob")
consensus.attest_block(block, "0xCarol")
consensus.attest_block(block, "0xDave")

# Check finalization
finalized, message = consensus.finalize_block(block)
print(f"Block Status: {message}")
```

#### 4. Slashing

Penalize malicious or negligent validators.

```python
class SlashingMechanism:
    def __init__(self, validators):
        self.validators = validators.copy()
        self.slashing_history = []
    
    def slash_for_double_signing(self, validator, penalty_percentage=100):
        """
        Slash validator for signing two conflicting blocks
        """
        if validator not in self.validators:
            return False, "Validator not found"
        
        stake = self.validators[validator]
        penalty = stake * (penalty_percentage / 100)
        
        self.validators[validator] -= penalty
        if self.validators[validator] <= 0:
            del self.validators[validator]
        
        self.slashing_history.append({
            'validator': validator,
            'reason': 'double_signing',
            'penalty': penalty
        })
        
        return True, f"Slashed {penalty} tokens (100% of stake)"
    
    def slash_for_inactivity(self, validator, penalty_percentage=1):
        """
        Slash validator for being offline
        """
        if validator not in self.validators:
            return False, "Validator not found"
        
        stake = self.validators[validator]
        penalty = stake * (penalty_percentage / 100)
        
        self.validators[validator] -= penalty
        
        self.slashing_history.append({
            'validator': validator,
            'reason': 'inactivity',
            'penalty': penalty
        })
        
        return True, f"Slashed {penalty} tokens (1% of stake)"
    
    def get_slashing_history(self):
        return self.slashing_history

# Example
validators = {"0xAlice": 32, "0xBob": 64, "0xMalicious": 128}
slashing = SlashingMechanism(validators)

print("Initial Stakes:")
for v, s in validators.items():
    print(f"  {v}: {s} ETH")

# Malicious validator double-signs
slashing.slash_for_double_signing("0xMalicious")

# Bob goes offline
slashing.slash_for_inactivity("0xBob")

print("\nAfter Slashing:")
for v, s in slashing.validators.items():
    print(f"  {v}: {s} ETH")

print("\nSlashing History:")
for event in slashing.get_slashing_history():
    print(f"  {event['validator']}: {event['reason']} - {event['penalty']} ETH")
```

### PoS Advantages

✅ **Energy Efficient**: 99% less energy than PoW
✅ **Lower Barrier to Entry**: No specialized hardware
✅ **Faster Finality**: Blocks finalized in minutes
✅ **Economic Security**: Slashing deters attacks
✅ **Scalability**: Better foundation for scaling solutions

### PoS Disadvantages

❌ **Nothing at Stake**: Validators could validate multiple chains (mitigated by slashing)
❌ **Rich Get Richer**: Higher stake = more rewards
❌ **Initial Distribution**: How to fairly distribute initial stake?
❌ **Long-Range Attacks**: Rewriting history from genesis (mitigated by checkpoints)
❌ **Less Battle-Tested**: Newer than PoW

### Ethereum's Transition to PoS

**The Merge (September 15, 2022)**

```
Before (PoW):
- Energy: ~100 TWh/year
- Issuance: ~13,000 ETH/day
- Hardware: GPUs

After (PoS):
- Energy: ~0.01 TWh/year (99.95% reduction)
- Issuance: ~1,600 ETH/day (88% reduction)
- Hardware: Standard computers
```

## Other Consensus Mechanisms

### Delegated Proof of Stake (DPoS)

**Used by**: EOS, TRON, Cosmos

**How it works:**
1. Token holders vote for delegates
2. Top N delegates become validators
3. Delegates rotate block production
4. Delegates earn rewards, share with voters

**Advantages:**
- Fast transaction finality
- High throughput
- Democratic governance

**Disadvantages:**
- More centralized (fewer validators)
- Voter apathy
- Cartel formation risk

```python
class DelegatedProofOfStake:
    def __init__(self, num_delegates):
        self.num_delegates = num_delegates
        self.votes = {}  # {delegate: total_votes}
        self.voters = {}  # {voter: {delegate: vote_amount}}
    
    def vote(self, voter, delegate, vote_amount):
        """
        Voter delegates their tokens to a delegate
        """
        if voter not in self.voters:
            self.voters[voter] = {}
        
        self.voters[voter][delegate] = vote_amount
        
        if delegate not in self.votes:
            self.votes[delegate] = 0
        self.votes[delegate] += vote_amount
    
    def get_active_delegates(self):
        """
        Return top N delegates by vote count
        """
        sorted_delegates = sorted(
            self.votes.items(),
            key=lambda x: x[1],
            reverse=True
        )
        return [d[0] for d in sorted_delegates[:self.num_delegates]]
    
    def rotate_block_producer(self, slot):
        """
        Rotate block production among active delegates
        """
        active_delegates = self.get_active_delegates()
        return active_delegates[slot % len(active_delegates)]

# Example: EOS-style DPoS with 21 delegates
dpos = DelegatedProofOfStake(num_delegates=21)

# Users vote for delegates
dpos.vote("user1", "delegate_A", 1000)
dpos.vote("user2", "delegate_A", 500)
dpos.vote("user3", "delegate_B", 800)
dpos.vote("user4", "delegate_C", 1200)
# ... more votes

active = dpos.get_active_delegates()
print(f"Active Delegates: {active[:5]}...")  # Top 5
```

### Proof of Authority (PoA)

**Used by**: VeChain, xDai Chain, private blockchains

**How it works:**
1. Pre-approved validators (known identities)
2. Validators take turns producing blocks
3. Reputation-based system

**Advantages:**
- Very fast and efficient
- Predictable block times
- Low computational requirements

**Disadvantages:**
- Centralized (permissioned)
- Requires trust in validators
- Not suitable for public blockchains

### Practical Byzantine Fault Tolerance (PBFT)

**Used by**: Hyperledger Fabric, Zilliqa

**How it works:**
1. Primary node proposes block
2. Nodes go through pre-prepare, prepare, commit phases
3. Block finalized when 2/3+ nodes agree

**Advantages:**
- Instant finality
- No forking
- Energy efficient

**Disadvantages:**
- Limited scalability (communication overhead)
- Requires known validator set
- Vulnerable if >1/3 nodes are malicious

### Proof of History (PoH)

**Used by**: Solana

**How it works:**
1. Cryptographic clock timestamps events
2. Proves time has passed between events
3. Combined with PoS for consensus

**Advantages:**
- Extremely high throughput (65,000 TPS)
- Low latency
- Efficient ordering

**Disadvantages:**
- Complex implementation
- Hardware requirements
- Network stability concerns

## Finality in Consensus

### Probabilistic Finality (PoW)

Confidence increases with each confirmation, but never 100% certain.

```
1 confirmation: ~90% confidence
6 confirmations: ~99.9% confidence (Bitcoin standard)
More confirmations: Higher confidence, never absolute
```

### Deterministic Finality (PoS, PBFT)

Once finalized, block cannot be reverted.

```
Ethereum PoS: 2 epochs (~13 minutes) for finality
Tendermint: Instant finality
Algorand: Instant finality
```

## Conclusion

Consensus mechanisms are the heart of blockchain technology, enabling decentralized agreement without trusted intermediaries. Each mechanism makes different trade-offs between decentralization, security, and scalability.

**Key Takeaways:**
- PoW: Battle-tested but energy-intensive
- PoS: Energy-efficient with economic security
- DPoS: Fast but more centralized
- PoA: Efficient for permissioned networks
- Hybrid approaches combine benefits

The choice of consensus mechanism depends on the specific requirements of the blockchain: public vs. private, speed vs. decentralization, energy efficiency vs. security guarantees.
