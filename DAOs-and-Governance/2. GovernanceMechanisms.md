# Governance Mechanisms

## Introduction

Governance mechanisms are the processes and systems through which DAOs make decisions, implement changes, and coordinate collective action. This document explores the technical implementation of various governance systems, voting mechanisms, proposal lifecycles, and security considerations.

## Proposal Lifecycle

### Standard Proposal Flow

```
1. Ideation → 2. Discussion → 3. Formal Proposal → 4. Voting → 5. Execution
```

### Detailed Stages

**1. Ideation Phase**
- Community members discuss ideas informally
- Gather initial feedback
- Refine concepts
- Build support

**2. Temperature Check**
- Off-chain signaling vote (Snapshot)
- Gauge community interest
- No gas costs
- Non-binding

**3. Formal Proposal**
- On-chain proposal submission
- Requires minimum token threshold
- Proposal details encoded
- Voting period begins

**4. Voting Period**
- Token holders cast votes
- Delegation active
- Quorum tracking
- Vote tallying

**5. Timelock Period**
- Delay before execution
- Security buffer
- Emergency cancellation window
- Final review

**6. Execution**
- Automated execution via smart contract
- Or manual execution by authorized party
- On-chain state changes
- Treasury operations

### Creating and Executing Proposals

```javascript
const { ethers } = require("hardhat");

async function createProposal() {
    const governor = await ethers.getContract("DAOGovernor");
    const token = await ethers.getContract("GovernanceToken");
    
    // Proposal: Transfer 1000 tokens from treasury
    const treasury = await ethers.getContract("Treasury");
    const recipient = "0x...";
    const amount = ethers.utils.parseEther("1000");
    
    // Encode function call
    const transferCalldata = treasury.interface.encodeFunctionData(
        "transfer",
        [token.address, recipient, amount]
    );
    
    // Create proposal
    const proposeTx = await governor.propose(
        [treasury.address],           // targets
        [0],                          // values
        [transferCalldata],           // calldatas
        "Proposal #1: Fund development team" // description
    );
    
    const proposeReceipt = await proposeTx.wait();
    const proposalId = proposeReceipt.events[0].args.proposalId;
    
    console.log(`Proposal created with ID: ${proposalId}`);
    return proposalId;
}

async function voteOnProposal(proposalId, support) {
    const governor = await ethers.getContract("DAOGovernor");
    
    // support: 0 = Against, 1 = For, 2 = Abstain
    const voteTx = await governor.castVote(proposalId, support);
    await voteTx.wait();
    
    console.log(`Voted ${support} on proposal ${proposalId}`);
}

async function executeProposal(proposalId, proposal) {
    const governor = await ethers.getContract("DAOGovernor");
    
    // Queue proposal in timelock
    const queueTx = await governor.queue(
        proposal.targets,
        proposal.values,
        proposal.calldatas,
        proposal.descriptionHash
    );
    await queueTx.wait();
    
    console.log("Proposal queued in timelock");
    
    // Wait for timelock delay
    await new Promise(resolve => setTimeout(resolve, 2 * 24 * 60 * 60 * 1000)); // 2 days
    
    // Execute proposal
    const executeTx = await governor.execute(
        proposal.targets,
        proposal.values,
        proposal.calldatas,
        proposal.descriptionHash
    );
    await executeTx.wait();
    
    console.log("Proposal executed");
}
```

### Snapshot + On-Chain Execution

Bridge off-chain votes to on-chain execution using SafeSnap.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SnapshotExecutor {
    address public dao;
    mapping(bytes32 => bool) public executedProposals;
    
    event ProposalExecuted(bytes32 indexed proposalId);
    
    modifier onlyDAO() {
        require(msg.sender == dao, "Only DAO");
        _;
    }
    
    function executeProposal(
        bytes32 proposalId,
        address[] memory targets,
        uint256[] memory values,
        bytes[] memory calldatas
    ) external onlyDAO {
        require(!executedProposals[proposalId], "Already executed");
        
        executedProposals[proposalId] = true;
        
        for (uint256 i = 0; i < targets.length; i++) {
            (bool success, ) = targets[i].call{value: values[i]}(calldatas[i]);
            require(success, "Execution failed");
        }
        
        emit ProposalExecuted(proposalId);
    }
}
```

## Voting Mechanisms

### Simple Majority

Proposal passes if For > Against.

```solidity
function _voteSucceeded(uint256 proposalId)
    internal
    view
    virtual
    returns (bool)
{
    ProposalVote storage proposalVote = _proposalVotes[proposalId];
    return proposalVote.forVotes > proposalVote.againstVotes;
}
```

### Supermajority

Requires higher threshold (e.g., 66%).

```solidity
function _voteSucceeded(uint256 proposalId)
    internal
    view
    virtual
    returns (bool)
{
    ProposalVote storage proposalVote = _proposalVotes[proposalId];
    uint256 totalVotes = proposalVote.forVotes + proposalVote.againstVotes;
    
    return proposalVote.forVotes >= (totalVotes * 66) / 100;
}
```

## Delegation System

### Vote Delegation

Allow token holders to delegate voting power.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Votes.sol";

contract DelegationToken is ERC20Votes {
    constructor() ERC20("DAO Token", "DAO") ERC20Permit("DAO Token") {
        _mint(msg.sender, 1000000 * 10 ** decimals());
    }
    
    // Delegate voting power to another address
    function delegate(address delegatee) public virtual override {
        _delegate(_msgSender(), delegatee);
    }
    
    // Delegate by signature (gasless)
    function delegateBySig(
        address delegatee,
        uint256 nonce,
        uint256 expiry,
        uint8 v,
        bytes32 r,
        bytes32 s
    ) public virtual override {
        require(block.timestamp <= expiry, "Signature expired");
        
        address signer = ECDSA.recover(
            _hashTypedDataV4(
                keccak256(
                    abi.encode(
                        keccak256("Delegation(address delegatee,uint256 nonce,uint256 expiry)"),
                        delegatee,
                        nonce,
                        expiry
                    )
                )
            ),
            v,
            r,
            s
        );
        
        require(nonce == _useNonce(signer), "Invalid nonce");
        _delegate(signer, delegatee);
    }
    
    // Get current voting power
    function getVotes(address account) public view virtual override returns (uint256) {
        return _checkpoints[account][_numCheckpoints[account] - 1].votes;
    }
    
    // Get historical voting power
    function getPastVotes(address account, uint256 blockNumber)
        public
        view
        virtual
        override
        returns (uint256)
    {
        require(blockNumber < block.number, "Not yet determined");
        return _checkpointsLookup(_checkpoints[account], blockNumber);
    }
    
    // Required overrides
    function _afterTokenTransfer(
        address from,
        address to,
        uint256 amount
    ) internal virtual override(ERC20Votes) {
        super._afterTokenTransfer(from, to, amount);
    }
    
    function _mint(address to, uint256 amount)
        internal
        virtual
        override(ERC20Votes)
    {
        super._mint(to, amount);
    }
    
    function _burn(address account, uint256 amount)
        internal
        virtual
        override(ERC20Votes)
    {
        super._burn(account, amount);
    }
}
```

## Governance Attacks and Mitigations

### Flash Loan Attack

**Attack:** Borrow tokens, vote, return tokens in same transaction.

**Mitigation:** Use snapshot-based voting power.

```solidity
// Vulnerable
function getVotingPower(address account) public view returns (uint256) {
    return token.balanceOf(account); // Current balance
}

// Secure
function getVotingPower(address account, uint256 blockNumber) 
    public 
    view 
    returns (uint256) 
{
    return token.getPastVotes(account, blockNumber); // Historical balance
}
```

### Governance Takeover

**Attack:** Accumulate majority of tokens to control DAO.

**Mitigations:**
- Quadratic voting
- Voting caps
- Time-weighted voting
- Multi-sig veto power (temporary)

```solidity
contract ProtectedGovernance {
    uint256 public constant MAX_VOTING_POWER = 10000; // 10% cap
    
    function getVotingPower(address account) public view returns (uint256) {
        uint256 balance = token.balanceOf(account);
        uint256 totalSupply = token.totalSupply();
        uint256 maxPower = (totalSupply * MAX_VOTING_POWER) / 100000;
        
        return balance > maxPower ? maxPower : balance;
    }
}
```

### Proposal Spam

**Attack:** Create many proposals to overwhelm governance.

**Mitigation:** Proposal threshold and deposit.

```solidity
contract AntiSpamGovernance {
    uint256 public proposalThreshold = 10000 * 10**18; // 10k tokens
    uint256 public proposalDeposit = 1000 * 10**18;    // 1k tokens
    
    mapping(uint256 => address) public proposalDepositor;
    
    function propose(...) external returns (uint256) {
        require(
            token.balanceOf(msg.sender) >= proposalThreshold,
            "Insufficient tokens"
        );
        
        // Lock deposit
        token.transferFrom(msg.sender, address(this), proposalDeposit);
        
        uint256 proposalId = _createProposal(...);
        proposalDepositor[proposalId] = msg.sender;
        
        return proposalId;
    }
    
    function executeOrCancel(uint256 proposalId) external {
        if (proposalSucceeded(proposalId)) {
            // Return deposit
            token.transfer(proposalDepositor[proposalId], proposalDeposit);
        } else {
            // Burn or redistribute deposit
            token.burn(proposalDeposit);
        }
    }
}
```

### Bribery and Vote Buying

**Attack:** Pay voters to vote certain way.

**Mitigations:**
- Secret voting (ZK proofs)
- Commit-reveal schemes
- Reputation-based voting

## Optimistic Governance

Execute proposals optimistically with veto period.

```solidity
contract OptimisticGovernance {

        address proposer;
        bytes32 actionHash;
        uint256 executionTime;
        bool executed;
        bool vetoed;
    }
    
    mapping(uint256 => Proposal) public proposals;
    uint256 public vetoThreshold = 10; // 10% can veto
    uint256 public optimisticPeriod = 3 days;
    
    function proposeOptimistic(
        address target,
        bytes memory data
    ) external returns (uint256) {
        uint256 proposalId = proposalCount++;
        
        proposals[proposalId] = Proposal({
            proposer: msg.sender,
            actionHash: keccak256(abi.encode(target, data)),
            executionTime: block.timestamp + optimisticPeriod,
            executed: false,
            vetoed: false
        });
        
        return proposalId;
    }
    
    function veto(uint256 proposalId) external {
        Proposal storage proposal = proposals[proposalId];
        require(!proposal.executed, "Already executed");
        require(block.timestamp < proposal.executionTime, "Too late");
        
        uint256 vetoVotes = getVetoVotes(proposalId);
```

## Governance Analytics

### Tracking Participation

```python
from web3 import Web3
import pandas as pd

class GovernanceAnalytics:
    def __init__(self, governor_address, web3_provider):
        self.w3 = Web3(Web3.HTTPProvider(web3_provider))
        self.governor = self.w3.eth.contract(
            address=governor_address,
            abi=governor_abi
        )
    
    def get_proposal_stats(self, proposal_id):
        """Get voting statistics for a proposal"""
        proposal = self.governor.functions.proposals(proposal_id).call()
        
        return {
            'proposal_id': proposal_id,
            'for_votes': proposal[3],
            'against_votes': proposal[4],
            'abstain_votes': proposal[5],
            'total_votes': proposal[3] + proposal[4] + proposal[5],
            'participation_rate': self.calculate_participation(proposal_id),
            'status': self.get_proposal_state(proposal_id)
        }
    
    def calculate_participation(self, proposal_id):
        """Calculate voter participation rate"""
        proposal = self.governor.functions.proposals(proposal_id).call()
        total_votes = proposal[3] + proposal[4] + proposal[5]
        
        snapshot_block = proposal[2]
        total_supply = self.token.functions.getPastTotalSupply(
            snapshot_block
        ).call()
        
        return (total_votes / total_supply) * 100
    
    def get_voter_history(self, voter_address):
        """Get voting history for an address"""
        events = self.governor.events.VoteCast.createFilter(
            fromBlock=0,
            argument_filters={'voter': voter_address}
        ).get_all_entries()
        
        return pd.DataFrame([{
            'proposal_id': e.args.proposalId,
            'support': e.args.support,
            'weight': e.args.weight,
            'block': e.blockNumber
        } for e in events])
    
    def get_delegate_stats(self):
        """Get delegation statistics"""
        # Query delegation events
        delegates = {}
        events = self.token.events.DelegateChanged.createFilter(
            fromBlock=0
        ).get_all_entries()
        
        for event in events:
            delegatee = event.args.toDelegate
            if delegatee not in delegates:
                delegates[delegatee] = {
                    'delegators': set(),
                    'voting_power': 0
                }
            delegates[delegatee]['delegators'].add(event.args.delegator)
        
        # Get current voting power
        for delegatee in delegates:
            delegates[delegatee]['voting_power'] = self.token.functions.getVotes(
                delegatee
            ).call()
        
        return delegates
```

- Balance between decentralization and efficiency
- Protection against governance attacks
