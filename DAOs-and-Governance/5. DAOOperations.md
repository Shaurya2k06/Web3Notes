# DAO Operations and Best Practices

## Introduction

Effective DAO operations require coordination across distributed teams, transparent processes, and efficient tooling. This document covers operational frameworks, working group structures, contributor management, compensation models, and coordination tools that enable DAOs to function effectively.

### Performance Management

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract ContributorPerformance {
    struct Contributor {
        address wallet;
        uint256 joinDate;
        uint256 totalContributions;
        uint256 reputationScore;
        bool active;
    }
    
    struct Contribution {
        address contributor;
        string description;
        uint256 value;
        uint256 timestamp;
        bool verified;
    }
    
    mapping(address => Contributor) public contributors;
    mapping(uint256 => Contribution) public contributions;
    uint256 public contributionCount;
    
    event ContributorAdded(address indexed contributor);
    event ContributionRecorded(uint256 indexed contributionId, address indexed contributor);
    event ReputationUpdated(address indexed contributor, uint256 newScore);
    
    function addContributor(address wallet) external {
        contributors[wallet] = Contributor({
            wallet: wallet,
            joinDate: block.timestamp,
            totalContributions: 0,
            reputationScore: 100, // Starting score
            active: true
        });
        
        emit ContributorAdded(wallet);
    }
    
    function recordContribution(
        address contributor,
        string memory description,
        uint256 value
    ) external returns (uint256) {
        require(contributors[contributor].active, "Contributor not active");
        
        uint256 contributionId = contributionCount++;
        contributions[contributionId] = Contribution({
            contributor: contributor,
            description: description,
            value: value,
            timestamp: block.timestamp,
            verified: false
        });
        
        emit ContributionRecorded(contributionId, contributor);
        return contributionId;
    }
    
    function verifyContribution(uint256 contributionId) external {
        Contribution storage contribution = contributions[contributionId];
        require(!contribution.verified, "Already verified");
        
        contribution.verified = true;
        
        Contributor storage contributor = contributors[contribution.contributor];
        contributor.totalContributions += contribution.value;
        contributor.reputationScore += contribution.value / 100; // Simple reputation increase
        
        emit ReputationUpdated(contribution.contributor, contributor.reputationScore);
    }
    
    function getContributorStats(address wallet) external view returns (
        uint256 joinDate,
        uint256 totalContributions,
        uint256 reputationScore,
        bool active
    ) {
        Contributor memory contributor = contributors[wallet];
        return (
            contributor.joinDate,
            contributor.totalContributions,
            contributor.reputationScore,
            contributor.active
        );
    }
}
```

## Compensation Models

### 1. Fixed Salary

Traditional employment model.

```python
class SalaryCompensation:
    def __init__(self, annual_salary, payment_frequency='monthly'):
        self.annual_salary = annual_salary
        self.payment_frequency = payment_frequency
    
    def calculate_payment(self):
        if self.payment_frequency == 'monthly':
            return self.annual_salary / 12
        elif self.payment_frequency == 'bi-weekly':
            return self.annual_salary / 26
        elif self.payment_frequency == 'weekly':
            return self.annual_salary / 52
    
    def total_cost(self, num_contributors):
        """Calculate total annual cost"""
        return self.annual_salary * num_contributors
```

### 2. Bounty System

Task-based compensation.

```solidity
contract BountySystem {
    struct Bounty {
        string description;
        uint256 reward;
        address creator;
        address assignee;
        uint256 deadline;
        bool completed;
        bool paid;
    }
    
    mapping(uint256 => Bounty) public bounties;
    uint256 public bountyCount;
    
    IERC20 public paymentToken;
    
    event BountyCreated(uint256 indexed bountyId, uint256 reward);
    event BountyAssigned(uint256 indexed bountyId, address indexed assignee);
    event BountyCompleted(uint256 indexed bountyId);
    event BountyPaid(uint256 indexed bountyId, address indexed recipient);
    
    function createBounty(
        string memory description,
        uint256 reward,
        uint256 deadline
    ) external returns (uint256) {
        // Transfer reward to contract
        paymentToken.transferFrom(msg.sender, address(this), reward);
        
        uint256 bountyId = bountyCount++;
        bounties[bountyId] = Bounty({
            description: description,
            reward: reward,
            creator: msg.sender,
            assignee: address(0),
            deadline: deadline,
            completed: false,
            paid: false
        });
        
        emit BountyCreated(bountyId, reward);
        return bountyId;
    }
    
    function assignBounty(uint256 bountyId, address assignee) external {
        Bounty storage bounty = bounties[bountyId];
        require(msg.sender == bounty.creator, "Only creator");
        require(bounty.assignee == address(0), "Already assigned");
        
        bounty.assignee = assignee;
        emit BountyAssigned(bountyId, assignee);
    }
    
    function completeBounty(uint256 bountyId) external {
        Bounty storage bounty = bounties[bountyId];
        require(msg.sender == bounty.assignee, "Only assignee");
        require(!bounty.completed, "Already completed");
        require(block.timestamp <= bounty.deadline, "Deadline passed");
        
        bounty.completed = true;
        emit BountyCompleted(bountyId);
    }
    
    function approveBounty(uint256 bountyId) external {
        Bounty storage bounty = bounties[bountyId];
        require(msg.sender == bounty.creator, "Only creator");
        require(bounty.completed, "Not completed");
        require(!bounty.paid, "Already paid");
        
        bounty.paid = true;
        paymentToken.transfer(bounty.assignee, bounty.reward);
        
        emit BountyPaid(bountyId, bounty.assignee);
    }
}
```

### 3. Coordinape (Peer-to-Peer Allocation)

Contributors allocate tokens to each other based on perceived value.

```python
class CoordinapeRound:
    def __init__(self, total_budget, participants):
        self.total_budget = total_budget
        self.participants = participants
        self.allocations = {p: {} for p in participants}
        self.give_tokens = {p: 100 for p in participants}  # Each gets 100 GIVE tokens
    
    def allocate(self, from_participant, to_participant, amount):
        """Allocate GIVE tokens from one participant to another"""
        if amount > self.give_tokens[from_participant]:
            raise ValueError("Insufficient GIVE tokens")
        
        self.give_tokens[from_participant] -= amount
        
        if to_participant not in self.allocations[from_participant]:
            self.allocations[from_participant][to_participant] = 0
        
        self.allocations[from_participant][to_participant] += amount
    
    def calculate_distribution(self):
        """Calculate final token distribution based on allocations"""
        total_received = {}
        
        for from_p in self.allocations:
            for to_p, amount in self.allocations[from_p].items():
                if to_p not in total_received:
                    total_received[to_p] = 0
                total_received[to_p] += amount
        
        total_gives = sum(total_received.values())
        
        distribution = {}
        for participant, received in total_received.items():
            distribution[participant] = (received / total_gives) * self.total_budget
        
        return distribution
```

### 5. Token Vesting

Long-term alignment through vested tokens.

```solidity
contract TokenVesting {
    struct VestingSchedule {
        uint256 totalAmount;
        uint256 startTime;
        uint256 cliffDuration;
        uint256 vestingDuration;
        uint256 claimed;
    }
    
    mapping(address => VestingSchedule) public vestingSchedules;
    IERC20 public token;
    
    event VestingCreated(address indexed beneficiary, uint256 amount);
    event TokensClaimed(address indexed beneficiary, uint256 amount);
    
    function createVesting(
        address beneficiary,
        uint256 totalAmount,
        uint256 cliffDuration,
        uint256 vestingDuration
    ) external {
        require(vestingSchedules[beneficiary].totalAmount == 0, "Vesting exists");
        
        token.transferFrom(msg.sender, address(this), totalAmount);
        
        vestingSchedules[beneficiary] = VestingSchedule({
            totalAmount: totalAmount,
            startTime: block.timestamp,
            cliffDuration: cliffDuration,
            vestingDuration: vestingDuration,
            claimed: 0
        });
        
        emit VestingCreated(beneficiary, totalAmount);
    }
    
    function claim() external {
        VestingSchedule storage schedule = vestingSchedules[msg.sender];
        require(schedule.totalAmount > 0, "No vesting schedule");
        
        uint256 claimable = getClaimableAmount(msg.sender);
        require(claimable > 0, "Nothing to claim");
        
        schedule.claimed += claimable;
        token.transfer(msg.sender, claimable);
        
        emit TokensClaimed(msg.sender, claimable);
    }
    
    function getClaimableAmount(address beneficiary) public view returns (uint256) {
        VestingSchedule memory schedule = vestingSchedules[beneficiary];
        
        if (block.timestamp < schedule.startTime + schedule.cliffDuration) {
            return 0; // Cliff not reached
        }
        
        uint256 elapsedTime = block.timestamp - schedule.startTime;
        
        if (elapsedTime >= schedule.vestingDuration) {
            return schedule.totalAmount - schedule.claimed; // Fully vested
        }
        
        uint256 vestedAmount = (schedule.totalAmount * elapsedTime) / schedule.vestingDuration;
        return vestedAmount - schedule.claimed;
    }
}
```

### Advice Process

```markdown
# ADVICE PROCESS

When making a decision:

## Step 1: Identify Decision Maker
- Person closest to the work
- Has context and expertise
- Will implement the decision

## Step 2: Seek Advice
Decision maker must seek advice from:
- People with expertise
- People who will be affected
- Relevant working group leads

## Step 3: Make Decision
- Decision maker considers all advice
- Makes final decision
- Not required to follow advice
- Must seriously consider it

## Step 4: Communicate
- Announce decision
- Explain reasoning
- Share advice received
- Document in appropriate channel

## Step 5: Implement
- Execute decision
- Monitor results
- Adjust if needed
- Share learnings
```

## Conflict Resolution

```markdown
# CONFLICT RESOLUTION PROCESS

## Level 1: Direct Communication
- Parties discuss directly
- Seek to understand perspectives
- Find mutually acceptable solution
- Document agreement

## Level 2: Mediation
If Level 1 fails:
- Request mediator from community
- Mediator facilitates discussion
- Helps parties find solution
- Non-binding recommendations

## Level 3: Working Group Review
If Level 2 fails:
- Escalate to relevant working group
- Working group reviews situation
- Makes recommendation
- May be binding depending on charter

## Level 4: Governance
If Level 3 fails or inappropriate:
- Create governance proposal
- Community discussion
- Token holder vote
- Binding decision

## Level 5: Arbitration
For legal disputes:
- Formal arbitration process
- As specified in operating agreement
- Legally binding
- Enforceable in courts
```
